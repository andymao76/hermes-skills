# 宠物健康管理与理赔工作流

## 适用场景
主人提供宠物医疗单据（病历、账单、检查报告），需要：
1. 归档到知识库形成完整健康档案
2. 给保险公司提交理赔申请摘要
3. 设置用药提醒（cron定时推送）

## 标准流程

### Step 1：提取PDF中的关键信息
```bash
pdftotext 文件.pdf - -layout
```
提取字段：
- 宠物名、种类、年龄、体重、性别
- 主人姓名、电话
- 医院名称、医生
- 诊断结论（初步诊断/最终诊断）
- 检查项目及费用明细
- 药品清单（名称、规格、数量、单价）
- 就诊日期

### Step 2：创建结构化健康档案
存放在 `knowledge/丢丢/` 或 `knowledge/宠物名/` 目录下，包含：
```
_index.md          ← 目录索引
丢丢健康档案.md     ← 完整档案（以下字段）
病情跟踪.md         ← 每日跟踪日志
照片/_index.md      ← 照片索引
```

健康档案必备字段：
```
🐾 宠物档案（名字/品种/性别/年龄/体重/主人）
🏥 就诊记录（医院/医生/诊断）
💰 费用明细（表格：项目|单价|数量|金额）
💊 用药方案（药品|用量|用法|费用）
👨‍⚕️ 医生信息（姓名/专长/单位）
📋 居家监测计划
🚨 警报信号
```

### Step 3：生成理赔申请摘要
按保险公司格式，三要素：
```markdown
就诊时间：YYYY年M月D日
患病情况：[宠物名]（X岁X品种，Xkg）因[主诉]就诊。
经[检查项目]确诊为[诊断]。
相关费用：诊疗费用共计XXX元，明细如下：
- [项目] XX.XX元
...
```

### Step 4：设置用药提醒cron
```bash
# 08:00 推送，40分钟后服药
cronjob create \
  schedule="0 8 * * *" \
  name="丢丢早上服药提醒" \
  prompt="☀️ 服药提醒！请在8:40前喂药：匹莫苯丹+呋塞米" \
  deliver=origin

# 20:00 推送，40分钟后服药
cronjob create \
  schedule="0 20 * * *" \
  name="丢丢晚上服药提醒" \
  prompt="🌙 服药提醒！请在20:40前喂药：匹莫苯丹+呋塞米" \
  deliver=origin
```

## 用户偏好
- 所有输出必须为**结构化表格**，不要纯文本段落
- 费用明细表格：项目|单价|数量|金额（含合计行）
- 理赔摘要：简洁三栏格式（就诊时间|患病情况|费用）
- 提醒时间与服药时间差40分钟（提醒→8:00/20:00，服药→8:40/20:40）

## 常见问题
- PDF提取失败 → 尝试 pdftotext -layout，fallback到 python PyPDF2/pdfminer
- 图片病历 → 用 Bailian Qwen3-VL-Plus 或 SiliconFlow Qwen3-VL 进行视觉识别
- 账单包含多张消费单 → 按消费单编号分组统计
