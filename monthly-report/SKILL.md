# 运维开发月报

生成适用于运维/开发人员的专业月报。

## 核心公式

核心数据汇总 + 亮点不足复盘 + 下月战略规划

## 撰写原则

1. **执行摘要先行**：3句话概括本月最值得被记住的亮点
2. **数据驱动呈现**：关键指标汇总，环比/同比对比
3. **深度复盘而非堆砌**：有总结和洞察
4. **问题转化为成长**：挑战→改进方向
5. **规划体现战略**：贴合公司/部门战略方向

## 输出格式

```
【月报】- {月份} - {姓名}

〖执行摘要〗
本月完成{核心成果}，实现{关键数据}。下月重点{核心目标}。

〖月度核心数据〗
{关键指标1}：{本月数值}（环比↑/↓X%）
{关键指标2}：{本月数值}（环比↑/↓X%）

〖重点项目进展〗
【项目A】进度：{完成度} | 关键节点：{里程碑} | 成果：{量化}

〖亮点与价值创造〗
{亮点1}：{突破/成果} → {业务/技术价值}

〖问题与不足复盘〗
{问题}：{根因} → {改进措施}

〖下月重点计划〗
P0: {核心目标 + 预期产出}
P1: {次要目标}

〖需支持与资源〗
```

## 岗位关注点

### 运维
- 稳定性成果：可用性提升、故障减少、MTTR下降
- 效率提升：自动化覆盖率提高、人力释放
- 容量与扩展性：集群扩容、性能优化
- 成本优化：云成本下降
- 安全合规：漏洞修复

### 开发
- 项目交付：里程碑、功能上线、交付质量
- 效率提升：开发效率优化、技术债务清理
- 技术突破：架构优化、QPS提升
- 质量保障：Bug率下降、测试覆盖率提升
- 知识沉淀：技术文档、经验分享

## 价值表达示例

- ❌ "巡检了10台服务器"
- ✅ "完成10台服务器日常巡检，发现并处理2个潜在风险，保障了平台稳定性"
- ❌ "写完支付模块代码"
- ✅ "完成支付模块开发并提前2天提测，测试覆盖率85%高于部门均值"

## 触发词

写月报、生成月报、月度汇报、monthly report
