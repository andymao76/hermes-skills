# 反模式案例与修正对比

> 本文件记录使用本 Skill 时常见的低效用法，并提供改进建议。

## 反模式 01：指令过于模糊

### ❌ 低质量输入
```
"帮我写个文章"
```

**问题：** 缺少主题、平台、长度、受众，输出将是通用泛化内容。

**✅ 正确做法：**
```
帮我写一篇关于 AI 工具的小红书种草文，1000字，目标读者是职场新人
```

---

## 反模式 02：省略业务背景

### ❌ 低质量输入
```
"做个促销方案"
```

**问题：** 没有产品类型、客单价、渠道等关键信息，方案无法聚焦。

**✅ 正确做法：**
```
做一个双十一促销方案，我卖手工皂，客单价 80 元，主渠道是微店
```

---

## 反模式 03：笼统的修改要求

### ❌ 低质量输入
```
"改一下"
```

**问题：** 不知道改什么方向，AI 只能猜测。

**✅ 正确做法：**
```
把这段文案的语气改得更年轻化，减少正式用词
```

---

## 反模式 04：一次提太多需求

### ❌ 低质量输入
一次提 10 个不相关的需求。

**✅ 正确做法：**
一次聚焦 1～2 个相关需求，效果更精准。

---

## 反模式 05：省略关键信息

### ❌ 低质量输入
只说要做什么，不说背景。

**✅ 正确做法：**
提供目标受众、平台、产品特点这三个关键信息。
