# A1 项目基础设施文档化模式

本文件记录了系统化文档化项目基础设施的方法论，适用于类似 A1 项目（苏丹谛听）的多组件、多站点运维场景。

## 文档化层级

```
项目背景 → 网络拓扑 → 组件清单 → 每组件详细运维 → 排查案例
```

### 1. 项目背景
- 项目名称、代号、客户
- 站点数、网段、访问方式
- 关键凭据（URL/用户/密码模板）

### 2. 网络拓扑
- 服务器 IP ↔ 主机名 ↔ 角色映射表
- 路由配置（Windows/服务器侧）
- 端口清单

### 3. 组件清单按依赖顺序
```
基础设施层: ZK → HDFS → YARN
存储计算层: HBase / Hive / Greenplum / JanusGraph / ES
消息层: Kafka / Redis
应用层: OWLS / ZTLIG / SICMS
监控层: Zabbix
```

### 4. 每个组件包含
- 连接方式（CLI / Web UI / API）
- 常用命令速查（增删改查）
- 启动/停止/重启命令
- 日志位置
- 常见问题（故障现象 → 原因 → 修复）

### 5. 排查案例
- 按业务场景组织（非按组件）
- 含完整排查链路（从现象到根因）
- 含验证步骤

## 项目专属标注规则

- 所有 IP、端口、Topic 名称、表名必须标注为项目专属
- 跨项目通用的命令（如 `hdfs fsck`）和项目特有参数（如 `215.152.1.15:9092`）分开写
- 使用 ⚠️ 标记强调项目特殊性

## 本技能的组织原则

- **SKILL.md**: 核心命令速查 + 故障排查流程（日常用）
- **references/**: 会话沉淀的深度分析、专用排查指南
- **知识库笔记**: 完整的技术手册，含完整命令输出和详细说明
- SKILL.md 指向知识库笔记的交叉引用，避免重复
