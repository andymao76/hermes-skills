## 一、GP（Greenplum）数据表

### 1. 目标管理

| 表名 | 作用 | 关键查询 |
|------|------|----------|
| lims_bds_case_group | Case group 基本信息 | `SELECT * FROM lims_bds_case_group ORDER BY case_group_id ASC;` |
| lims_bds_case | 案件基本信息 | `SELECT * FROM lims_bds_case WHERE case_name = '...' ORDER BY group_id ASC;` |
| lims_bds_case_casegroup | Casegroup 和 Case 关系 | `SELECT * FROM lims_bds_case_group;` |
| lims_bds_case_user | Case 和操作员关系 | `SELECT * FROM lims_bds_case_user ORDER BY case_id ASC;` |
| **lims_bds_group_target** | **Target 基本信息（核心）** | `SELECT * FROM lims_bds_group_target WHERE target_id='...' / map_id='...';` |
| lims_bds_target | 目标账号 | `SELECT * FROM lims_bds_target WHERE target_id='...';` |
| lims_bds_target_office | Target 和 Office 关系 | `SELECT * FROM lims_bds_target_office WHERE status='7';` |
| lims_bds_case_target | Target 和 Case 关系 | `SELECT * FROM lims_bds_case_target ORDER BY case_id ASC;` |
| lims_bds_target_user | Target 和 Operator 关系 | `SELECT * FROM lims_bds_target_user ORDER BY map_id ASC;` |
| lims_bds_target_user_focus | 用户感兴趣的 Target | `SELECT * FROM lims_bds_target_user_focus ORDER BY user_id ASC;` |
| rls_hts_count | 昨日目标命中统计 | `SELECT * FROM rls_hts_count ORDER BY hts_cnt DESC LIMIT 10;` |
| hts_clue_hit | 目标短暂命中 | `SELECT * FROM hts_clue_hit LIMIT 10;` |

### 2. 目标查询 & 字段配置

| 表名 | 作用 |
|------|------|
| lims_trs_state | 列表状态 |
| lims_til_level | 标签层次结构（原名 lims_to_level） |
| rds_source_info | 数据源定义 — 所有 sourceno |
| rds_query_define | 查询显示字段定义 — 不同协议字段映射 |
| rds_query_showfield | 字段显示配置 — 按账号配置显示/导出/顺序 |
| rds_source_define | 命中数据详情+列表字段配置 |
| **hts_lig_hi2** | **TMC 命中表（最核心！）** |

> `hts_lig_hi2` 核心字段: tid, clue_id, msisdn, imsi, imei_esn_meid, capturetime, site_id, sourceno, file_path
> 
> sourceno 映射:
> - DST_014031 → 通话 (service_type=1)
> - DST_014001 → SMS (service_type=3)
> - DST_014032 → 位置 (service_type=0)
> - DST_CDR → 通话协议

关键查询：
```sql
SELECT tid, clue_id, msisdn, imsi, imei_esn_meid, capturetime, site_id, sourceno
FROM hts_lig_hi2
WHERE tid = '...' AND clue_id = ...;
-- 按时间范围
SELECT * FROM hts_lig_hi2
WHERE clue_id IN (...) AND capturetime BETWEEN 1750839600 AND 1750843199;
```

## 三、数据导出方法

### GP 导出到 CSV
```bash
timestamp=$(date +%Y%m%d%H%M)
psql -h 215.152.1.13 -p 5432 -U daedb -d bigdata \
  -c "\\copy (SELECT * FROM hts_lig_hi2 LIMIT 10) TO './hts_lig_hi2_${timestamp}.csv' WITH CSV HEADER"
```

### Hive 导出到 CSV
```bash
beeline -u jdbc:hive2://215.152.1.11:10000/zxsk_lis \
  --showHeader=true --outputformat=csv --nullemptystring=true \
  -e "select * from trs_hi3_ipdr where date_partition>='2025062310' limit 1" \
  1>trs_hi3_ipdr.csv 2>>/dev/null
```
