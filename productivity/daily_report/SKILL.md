# 日报生成 (daily_report)

从工作日志中读取当日记录，按格式生成日报。

## 触发方式

- `生成今天的日报`
- `日报用列表格式，不要表格`
- `日报加上明日计划`

## 使用节奏

每天下班前执行：`生成今天的日报`

## 存储路径

| 入口 | 路径 |
|------|------|
| Obsidian 原生 | `~/Documents/Obsidian Vault/工作/日报/YYYYMMDD.md` |
| kb-search 索引 | `~/knowledge/工作/日报/YYYYMMDD.md`（symlink） |
| 日期格式 | YYYYMMDD（如 20260611） |

生成后执行 `kb-search.py refresh` 刷新索引。

## ⚠️ 必填区段

用户硬性要求：每一份日报都必须包含以下三个区段，不可省略。使用 session_search 搜索当天所有会话（不限于当前会话），获取完整工作记录。

### 1. 软件项目

记录当天所有软件开发项目的进展。**用户明确要求软件项目工作必须与运维工作并列记录，不可遗漏。**

包括但不限于：
- 功能开发（新增/修改/重构内容）
- Bug 修复（问题现象 → 根因 → 修复方法）
- 测试结果（测试数量、通过率、回归测试）
- 代码仓库操作（Git 初始化、GitHub 推送、Fork）

格式：
```markdown
## 软件项目

### 1. 项目名

**工作内容：**
- 功能开发 ...
- 问题修复 ...
- 测试结果：N/N 通过 ✅
```

### 2. 日常维护

记录当天的运维工作，包括但不限于：
- 协助排查问题（问题现象 → 排查过程 → 结论）
- 远程巡检结果（各平台功能状态、告警情况）
- 项目跟踪进展（对接进度、配置备份等）
- 异常情况记录

格式：
```markdown
## 日常维护

### 1. 问题标题

**现象：** ...
**排查过程：** ...
**结论：** ...

### 2. 远程巡检
- **平台A：** 功能正常 / 异常
- ⚠️ **告警：** 已/未通知客户

### 3. 项目跟踪
- ...
```

### 3. 项目部署

记录当天的部署内容。**如果当天无部署任务，必须显式写明：**

```markdown
## 项目部署

（今日无项目部署任务）
```

不得省略此区段。

## 可选格式调整

| 用户说 | 效果 |
|--------|------|
| `日报用列表格式` | 表格转列表 |
| `加上今日总计工时` | 显示总工时 |
| `按工时排序` | 从多到少排序 |
| `加上明日计划` | 追加明日计划区块 |
| `不要明日计划` | 去掉明日计划 |

## 单项目日报模板

当默认项目（A1 PC项目）为主时，输出简化版：

```markdown
---
date: YYYYMMDD
tags: [日报]
project_hours:
  a1-pc-project: Xh
total_hours: Xh
---

# 日报 YYYYMMDD（周X）

### 项目：A1 PC项目（苏丹NISS）

| 工时 | 工作内容 |
|------|----------|
| Xh | ... |

**今日总计工时：Xh**

## 日常维护

### 1. ...

## 项目部署

（今日无项目部署任务）

## 明日计划

- [ ] ...
```

## 多项目日报模板

当有多个项目时，显示完整版：

```markdown
| 项目 | 工时 | 工作内容 |
|------|:----:|----------|
| A1 PC项目 | Xh | ... |
| 印尼项目 | Xh | ... |
```

## 工作流

1. **前置处理（本地脚本）** — `daily-report-prep --date YYYYMMDD` → 获取当日结构化数据（预聚合的项目+工时+条目）
2. 从 worklog 记录中读取当天所有记录
3. **无日志时回退**：若 `daily-report-prep` 返回无数据（如 `{"error": "未找到 ... 的日志文件"}`），用 `session_search(query="20260630", limit=5, sort="newest")` 搜索当天所有会话，从会话摘要中提取工作记录
4. 按配置格式生成日报
5. **确认补充日常维护和项目部署两部分**（必填）
6. 写入 `工作/日报/YYYYMMDD.md`
7. 确保 `~/knowledge/工作/日报/YYYYMMDD.md` 与 vault 内容一致（通过 symlink 或直接写入）
8. 刷新 kb-search 索引：`cd ~/knowledge && python3 ~/.hermes/scripts/kb-search.py refresh`

> 前置步骤完全本地运行，零 API 调用。脚本 `~/.local/bin/daily-report-prep` 自动聚合项目工时、分组条目，减少 LLM 读取原始日志的 token 消耗。

## cron 任务输出格式

作为 cron 作业运行时（无用户在场），最终响应输出结构化提醒而非写入对话：

```markdown
📋 **今日日报提醒 — YYYY-MM-DD（周X）**

共记录 {N} 项工作，合计约 {Xh} 可追踪工时。

| 时间 | 项目 | 工时 | 工作内容 |
|------|------|:----:|----------|
| ... | ... | ... | ... |

**今日新增知识库条目：{N} 篇** ✅ kb-search索引已刷新

⚠️ **请补充确认：**
- 工时期望调整？
- 是否有其他软件项目工作需要补充？
```

若当天无工作记录 → 输出 `[SILENT]`
