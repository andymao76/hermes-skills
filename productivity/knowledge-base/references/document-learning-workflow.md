# 文档学习 → 知识笔记 工作流

当用户提交文档（附件、粘贴内容、URL）说「学习这个」「放到知识库里」，执行以下标准流程。

## 完整工作流

```
1. 读取全文（大文件分段读取，确保覆盖全部章节）
       ↓
2. 提取核心结构（章节、关键规则、速查表）
       ↓
3. 浓缩归纳（保留关键信息，去除冗余例子和格式噪音）
       ↓
4. 写入知识库（YAML frontmatter + 结构化 Markdown + wikilinks）
       ↓
5. 双向交叉引用（patch 已有笔记补充 wikilinks）
       ↓
6. enzyme refresh 更新语义索引
       ↓
7. 验证文件大小和行数
```

## 第1步：读取全文

- 小文件 (<500行) 一次读完
- 中等文件 (500~5000行) 用 read_file 分段读取，offset 每次递增 500
- **超大文件 (>5000行)**：先读前 200 行获取目录结构 → 跳读中段（如 offset=总行数/2）感受内容密度 → 跳读末段（如 offset=总行数-200） → 确认 `truncated: false` → 基于目录结构直接提炼，不必逐段通读全文
- 跳过图片引用路径（如 `file:///C:\\Users\\...` ）和格式噪音
- 关注：章节标题、规则编号、关键词（规则/建议/禁止/必须）

**超大文件阅读示例**（20577 行 Java 规范）：
```
read_file(offset=1, limit=200)     → 获取目录结构
read_file(offset=6000, limit=200)  → 了解内容密度
read_file(offset=20400, limit=200) → 确认结尾结构
→ 基于目录直接归纳，无需逐段通读
```

## 第2步：提取核心结构

识别文档的骨架：
- 章节目录（1级/2级标题）
- 规则分类（如：布局/注释/命名/函数/可靠性/可测试性）
- 规则层级（规则 = 必须遵守，建议 = 推荐遵守）
- 速查表（表格形式的规则汇总）

## 第3步：浓缩归纳

**原则：**
- 原始文档 N 千行 → 笔记控制在 300 行以内（比例约 10:1 ~ 15:1）
- 保留所有关键规则编号和核心要求
- 去除冗余：重复的例子、格式噪音、无效图片引用、过多的正反例对比
- 保留代表性正反例（每种类型最多1个）
- 用表格替代段落描述（用户偏好表格）

**输出结构模板：**

```markdown
---
title: <文档标题>
tags:
  - <领域标签1>
  - <领域标签2>
related:
  - "[[已有笔记1]]"
  - "[[已有笔记2]]"
created: <YYYY-MM-DD>
source: <原始文件路径>
aliases:
  - <英文别名>
---

# <标题>

> 来源/作者/日期等元信息

## 一、文档结构概览

| 章节 | 主题 | 核心内容 |
|------|------|----------|
| ... | ... | ... |

## 二、关键规则速查（按主题分节）

...

## 关联笔记

- [[相关笔记1]]
- [[相关笔记2]]
```

## 第4步：写入知识库

### 目录选择规则

| 文档类型 | 目录 |
|----------|------|
| 编程规范/编码标准 | `~/knowledge/编程规范/` |
| 电信/3GPP/LI协议 | `~/knowledge/research/` |
| 数据库/中间件对比 | `~/knowledge/数据库相关/` |
| AI/Agent/LLM | `~/knowledge/AI_Agent相关/` |
| 运维/部署/系统 | `~/knowledge/notes/` |
| 通用技术调研 | `~/knowledge/research/` |

### YAML frontmatter 必填字段

```yaml
---
title: <中文标题>
tags: [<至少2个标签>]
created: <YYYY-MM-DD>
source: <原始文件路径>
---
```

可选字段：`aliases`（英文别名）、`related`（wikilinks 列表）

### 命名规范

- 文件名用中文，与标题一致
- 示例：`中新赛克C语言编码规范.md`、`Neo4j_vs_NebulaGraph_图数据库选型对比.md`

## 第5步：双向交叉引用

写入新笔记后，**必须**补充已有笔记的双向链接：

```bash
# 用 patch 工具为已有笔记的 frontmatter 添加 related 字段
patch(
  path="/home/andymao/knowledge/<目录>/<已有笔记>.md",
  old_string="related:\n  - \"[[笔记A]]\"",
  new_string="related:\n  - \"[[笔记A]]\"\n  - \"[[新笔记]]\""
)
```

**原则**：新笔记应链接 2-3 个已有笔记，已有笔记也应反向链接新笔记。这样 Obsidian 图谱才能形成完整网络。

## 第6步：索引更新

```bash
cd /home/andymao/knowledge && enzyme refresh
```

## 第7步：验证

```bash
wc -l /home/andymao/knowledge/<目录>/<文件名>.md
ls -lh /home/andymao/knowledge/<目录>/<文件名>.md
```

报告结果给用户：原始行数 → 笔记行数、文件大小、存储位置。

## 常见陷阱

- **不要逐段回应**：用户说「学习」意味着一次总结完，不要边读边评论
- **不要照搬原文**：用户要的是浓缩的结构化笔记，不是副本
- **不要遗漏 enzyme refresh**：索引不更新，知识库搜索不到新笔记
- **不要用 read_file 逐行通读超大文件**：>5000行先读目录+跳读代表段，基于目录归纳
- **大文件要确认读全**：必须看到 `truncated: false` 才停止读取
- **中文文件名**：笔记文件名用中文，与 Obsidian 图谱显示一致
- **写完新笔记要补双向链接**：patch 已有笔记的 frontmatter 加 `related` 字段
