# 任务闭环周期（第二大脑核心）

第二大脑的本质不是存得多，而是**闭环快**。

## 完整闭环

```
输入 → 理解 → 归档 → 提醒 → 执行 → 回顾 → 总结
```

## 各阶段在 Hermes 中的实现

| 阶段 | 实现 | 工具/位置 |
|------|------|-----------|
| **输入** | 所有外部信息先进 00_INBOX | 各通道消息、网页、文件 |
| **理解** | Hermes 分析内容，提取要点+待办 | AI 引擎 |
| **归档** | 按 PARA 分类存入知识库 | `kb-search.py refresh` |
| **提醒** | 每日 19:00 检查 inbox，推送到 Telegram | cron |
| **执行** | 用户处理 | — |
| **回顾** | 日报汇总当日完成 | `daily/YYYY-MM-DD.md` |
| **总结** | 周报复盘 + 更新 project_status.yaml | 周日 cron |

## 每日闭环（cron 21:00）

1. 检查 `00_INBOX/日报/今日` 是否存在
2. 若无 → 推送到 Telegram 问"今天做了什么"
3. 若有 → 提取待办事项，更新 project_status.yaml
4. 生成明日计划
5. 归档到 `daily/YYYY-MM-DD.md`

## 周度闭环（cron 周日 21:00）

1. 汇总本周所有日报
2. 统计完成事项/新增知识/故障次数
3. 更新 project_status.yaml
4. 生成下周计划
5. 推送到 Telegram + 飞书
