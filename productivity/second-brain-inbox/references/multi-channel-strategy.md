# 多通道策略（第二大脑 V5）

每个通道有明确用途，不是简单重复推送。

## 通道角色定义

| 通道 | 角色 | 用途 | 推送内容 |
|------|------|------|---------|
| **Telegram** | 个人主控台 | 日常交互、查询、指令 | 日报摘要、重要通知 |
| **飞书** | 工作输入口 | 企业协作、会议纪要 | 工作相关通知 |
| **微信** | 轻量提醒 | 被动接收、不互动 | 天气、关键预警 |
| **CLI** | 深度操作台 | 技术操作、调试、架构 | 无推送，主动访问 |
| **Discord** | 归档/备用 | 推送历史留存、备用通道 | 所有推送的归档副本 |

## 推送规则

- 重要通知 → 全通道推送
- 日报/周报 → Telegram + 飞书
- 天气/定时提醒 → 微信（轻量）
- CLI 始终是最高权限通道（不受限速影响）

## 避免的问题

- 消息到处散 → 各通道有明确边界
- 重复推送 → cron deliver 按角色配置
- 重要信息找不到 → 所有推送最终归入 knowledge/00_INBOX/
