# 工作日志 (worklog)

记录工作流水账，自动解析项目名称、工时、工作内容。

## 触发方式

### 单项目模式（默认项目 = A1 PC项目）

不写项目名时自动归到默认项目：

- `开始任务：整理结项文档` → A1 PC项目
- `结束任务，用了1.5小时` → 自动关联上一个开始的任务
- `记一下：准备验收材料，2小时` → A1 PC项目
- `补记昨天下午 客户沟通 1小时` → 自动补到昨日

### 多项目模式（指定项目名）

有多个项目时显式指定：

- `开始任务：印尼项目，需求分析`
- `结束任务，用了2小时`
- `记一下：下午3-4点处理埃及项目的数据问题`
- `补记昨天上午：9-11点写月报`

## 默认项目

当前默认项目：**A1 PC项目（苏丹NISS）**

不写项目名时自动归到此项目。详见项目映射表。

## 存储

日志写入 `工作/日报/YYYYMMDD.md`

> 完整路径: `~/Documents/Obsidian Vault/工作/日报/YYYYMMDD.md`  
> kb-search 索引: `~/knowledge/工作/日报/YYYYMMDD.md`（symlink）

## 使用节奏

| 时间 | 操作 | 命令 |
|------|------|------|
| 每天下班前 | 生成日报 | `生成今天的日报` |
| 每周五下班前 | 生成周报 | `生成本周周报` |
| 每月最后一天 | 生成月报 | `生成本月月报` |

## 日志格式

每条记录由 Hermes 自动整理为：
```
{时间戳}|{项目}|{工时}|{工作内容}
```

## 工作流

1. 用户输入工作记录，自动解析项目、工时、内容
2. 如信息不全 → 追问补充
3. 追加到当日日志（支持补记历史记录）
4. 每天下班前调用 daily_report skill 生成完整日报
