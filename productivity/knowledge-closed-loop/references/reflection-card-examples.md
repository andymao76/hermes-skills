# 反思卡片示例集

本文件记录了知识闭环系统 Phase 1 测试时生成的示例反思卡片，供后续参考。

## 示例 1：系统恢复全面诊断

**来源：** 用户问"系统恢复了吗" → 全面检查 Gateway/Clash/MCP/7个IM平台/cron

**关键发现：**
- 系统恢复诊断有标准流程：Gateway → Clash → 系统负载 → MCP → IM平台 → cron
- IM 连通性测试要发真实消息验证，不能只看进程状态
- Gateway 重启导致凌晨 cron 投递超时是临时问题

**结论：** ⭐ 升级为 Skill — 系统恢复诊断有固定检查清单

## 示例 2：百度网盘照片批量下载与 AI 视觉识别

**来源：** 用户需要从百度网盘找回宠物照片，用 Qwen-VL 批量识别

**关键发现：**
- 百度网盘有两种获取途径：bypy（简单但受限）vs 百度开放 API（灵活但需处理 token/dlink）
- Qwen-VL-Plus 批量分析照片内容，准确率不错
- 入库路径规范：文件名加年份前缀避免冲突

**结论：** 📝 存入 Knowledge — 使用频率低，适合作为技术笔记

## 示例 3：腾讯云远程安装 Hermes Agent

**来源：** 用户要求将技能同步到腾讯云 — 远程 SSH 安装失败

**关键发现：**
- 腾讯云对 GitHub HTTPS 的 TLS 连接不稳定（GnuTLS recv error -110）
- 两条修复路径：通过代理拉取代码 / 从本地 scp 源文件过去
- 远程安装失败的主要模式：GitHub 网络问题导致 clone 失败

**结论：** 📝 存入 Knowledge — 首次出现，踩坑记录

---

## 示例 4：GitHub API 认证失效的降级策略

**来源：** cron 定时任务 GitHub 生态搜索返回 401

**关键发现：**
- 检测 401 → 自动切换策略：MCP_GitHub → web_search → web_extract 三级降级
- 降级后数据要标注"近似值"，避免误导
- 失败模式可检测，不要一直重试

**结论：** ⭐ 升级为 Skill — GitHub API 降级是通用模式
