# 知识闭环系统架构

## 完整闭环流程图

```
问题 ──→ 解决 ──→ 经验总结 ──→ 反思卡片 ──→ 分流决策
                       ↑                         │
                       │               ┌─────────┴─────────┐
                       │               ▼                   ▼
                       │         ⭐ Skill              📝 Knowledge
                       │     (过程记忆)               (陈述记忆)
                       │               │                   │
                       │               ▼                   ▼
                       │     skill_manage()        存入知识库对应分类
                       │     + 备份到知识库         + 酶索引更新
                       │               │                   │
                       └─────── 未来复用 ←──────────┘
                                          │
                                          ▼
                                    cron 任务 / 用户提问
                                    → Skill 自动触发
                                    → Knowledge 可搜索
```

## 三层记忆架构

| 层 | 位置 | 内容 | 生命周期 |
|----|------|------|---------|
| 🧠 **Memory** | Hermes memory 系统 | 用户偏好、环境事实、工具惯用法 | 跨会话 |
| 💡 **Skill** | `~/.hermes/skills/` | 可执行的工作流、命令序列、决策规则 | 永久（可更新） |
| 📚 **Knowledge** | `~/knowledge/` + Obsidian | 技术笔记、参考文档、项目背景 | 永久 |

## 阶段路线图

### Phase 1 ✅（当前）
- 问题解决后由 Agent 主动生成反思卡片
- 人工审核后再升格为 Skill / Knowledge
- 卡片存在 INBOX 等待处理

### Phase 2（规划）
- 每天清晨 cron 扫描 INBOX 反思卡片
- 对无人处理的卡片做批量建议
- Agent 自动执行升格（需设定规则）

### Phase 3（规划）
- 冷启动：扫描历史 Session 中的故障排查
- 提取未沉淀的经验自动生成卡片
- 统计"Skill 命中率"作为闭环质量指标


## 关键词

- **反思卡片（Reflection Card）** — 每一次问题解决后的经验提取产物
- **闭环率** — 沉淀为 Skill/Knowledge 的问题数 / 总可沉淀问题数
- **Skill 命中率** — 已创建的 Skill 在后续任务中被触发的比例
