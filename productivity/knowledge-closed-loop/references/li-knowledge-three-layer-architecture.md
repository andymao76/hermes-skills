## 关键工作流

### 1. 知识口述采集（用户分段输出）

```
用户: "在华为SVC上监听，设控基本同CS模式..."
  → Agent: 查知识库，整理已接收内容为表格，询问继续
用户: "X2简析，基于IMS标准拦截..."
  → Agent: 追加到 X2 章节，patch 增量更新
用户: "iMS-Gen-IRI-Report 参数..."
  → Agent: 编表（枚举值表、参数表）
...
```

### 2. 原始协议数据处理

```
用户提供完整 X2 报告解码（13条消息）
  → Agent:
    ① 秩序重建（按时间排序）
    ② 提取公共字段（LIID/CallID/ICID 不变）
    ③ 时序归纳（13行流程表+对比表）
    ④ 观察总结（10条规则提炼）
```

### 3. 变更累积与分层

随着会话推进，知识体量变大后自然分为三层：
- 第1-9轮口述 → 方案原理篇

## 配置参数表记录模式

当用户分段提供 ZTLIG/ztlig.cfg 这类系统配置参数时（每条约10-25项，跨多轮次），采用以下模式：

### 配置表三栏 → 六栏扩展

用户的原始数据通常是 3列（名称/含义/取值），处理时扩展为 6列标准表：

```
| 配置项名称 | 类型 | 默认值 | 含义 | 取值范围 | 需重启进程 | 属性 |
```

| 列 | 作用 |
|-----|------|
| 配置项名称 | 保持原样（如 ztlig.ztlig2.x.networkType）|
| 类型 | 推断类型（整数/字符串/IP等）|
| 默认值 | 提炼默认值 |
| 含义 | 精简用户原文说明，必要时用括号补充关键细节 |
| 取值范围 | 枚举或范围值，如 [1-100] / [0-1] / 50000/100000等 |
| 需重启进程 | 从用户原始数据中提取 |
| 属性 | C=必配, M=重要, O=可选 |

### 分类方式

按配置块分节，每节一个 H3 标题：

```
### NE-COM 部分（NE 通用配置）
### NE-HW 部分（华为网元特有配置）
### NE-ZTE 部分（中兴网元特有配置）
### SSF 部分
### RVF 部分
```

### 追加模式

用户提供顺序不按章节，每轮只给一个模块。用 patch 追加到已有笔记的对应位置，保持结构最终统一。

### 注意事项

- 枚举值范围较宽或特殊值需在表后加 Notes 注明
- 配置项间的关联性（如 csmcnb/psmcnb 只有一个有效）在表后单独说明
- 最终给用户一个结构树概览

## 可复用原则

- ✅ **先查再写** — 每段口述前或批量入库前，先查知识库避免重复
- ✅ **随行编译** — 口述过程中持续展示结构化草稿，让用户确认方向
- ✅ **分批确认** — 长会话中段性告知当前已保存内容 + 剩余待补充
- ✅ **最终总览** — 所有笔记写入后给出拓扑表，方便用户导航
- ✅ **配置参数分类归档** — 同一系统的 config 按模块分节，保持表结构一致，便于后续章节间交叉引用
