# SVG 坐标批量偏移修复（Overlap 修正）

> 当 Layer 标签与底层盒子重叠时，通过批量偏移所有下层 Y 坐标解决。

## 适用场景

- Layer 标签条与上一层底部盒子重叠（如标签 y=150，上层盒子底部 y=205）
- 新增或删除层后需要整体平移下方内容
- SVG 重构需要统一偏移

## 核心原则

不要仅平移标签条——必须同步平移该层及其后的所有元素（盒子、连线、箭头、路径），否则下层会与标签条继续重叠。

## 修复工作流

### 第一步：确定偏移量

```
重叠量 = 标签条顶部 - 上层盒子底部
安全偏移量 = 重叠量 + 至少 5px 间距
```

示例（本次修复）：
- Layer 1 盒子底部: y=150+55=205
- Layer 2 标签顶部: y=150（重叠）
- 偏移量: +60px（标签移至 y=210，留 5px 间距）

### 第二步：分层写入偏移

不要用脚本一次批量替换全部 y 值——容易误伤不需要偏移的元素（如图例、Layer 1 内部的箭头等）。**逐层手动 patch 更可靠。**

```
# 正确的偏移模式：每层独立 patch
- Layer N 标签条:       y += 偏移量   (1 个 rect)
- Layer N 标签文字:      y += 偏移量   (1 个 text)
- Layer N 所有盒子:      y += 偏移量   (N 个 rect + 对应 text)
- Layer N → N+1 箭头:    y1 += 偏移量, y2 += 偏移量
- Layer N+1 标签条及以下: y += 偏移量   (重复)
```

### 第三步：更新 viewBox 高度

```
新 viewBox 高度 = 最大元素底部 + 50px 边距
```

### 第四步：更新背景网格

SVG 中用绝对尺寸替换百分比：`<rect width="100%" height="100%">` → `<rect width="1340" height="1100">`

## 实例：Hermes 架构图修复 (V4.1→4.2)

### 问题描述
Layer 2 标签条 `y=150` 与 Layer 1 底部 CLI/API Server 盒子 `y=150~205` 完全重叠。

### 修复步骤

1. **Layer 2 标签条：** `y=150→210`（+60），标签文字 `y=167→227`
2. **Layer 2 盒子及连线：** `y=220→260`（+40），对应文字同步偏移
3. **Layer 3–7：** 所有 y 统一 +40
4. **viewBox：** 1040 → 1100
5. **背景网格 rect：** `height="1040"` → `height="1100"`

### 各层偏移对照

| 元素 | 原 y | 新 y | 偏移量 |
|------|------|------|--------|
| Layer 2 标签 | 150 | 210 | +60 |
| Layer 2 盒子 | 220 | 260 | +40 |
| Layer 3 标签 | 300 | 340 | +40 |
| Layer 3 盒子 | 325 | 365 | +40 |
| Layer 4 标签 | 420 | 460 | +40 |
| Layer 5 标签 | 620 | 660 | +40 |
| Layer 6 标签 | 820 | 860 | +40 |
| Layer 7 标签 | 940 | 980 | +40 |
| Layer 7 盒子 | 975 | 1015 | +40 |

### 注意

- **Layer 2 为什么偏移 +60 而不是 +40？** 因为需要与 Layer 1 底部（y=205）拉开距离，从 y=150 到 y=210 留了 5px 间隙
- **其余层为什么只 +40？** 保持 Layer 2→3 之间的原始间距不变
- **箭头同步检查：** 每个箭头 (`y1`/`y2`) 都必须同步偏移，否则会出现箭头断连或指向空白
