# 知识图谱迭代工作流

使用 architecture-diagram skill 创建知识图谱时的迭代模式。

## 典型迭代流程

```
v1.0 → 初版结构 → 用户反馈缺失内容
v2.0 → 新增分类 → 用户反馈布局重叠
v2.1 → 修正间距/标签 → 用户反馈命名错误
...
```

## 每次迭代必须做的检查

### 1. 内容完整性
- 所有一级分类是否存在？
- 所有二级/子分类是否覆盖？
- 有无遗漏的厂商/技术域？
- 命名是否准确？（如 NISS→SICMS、LI→L1 等术语错误）

### 2. 布局正确性
用布局公式重新计算所有框体位置，不要只靠目测微调：

```
框数 N，总宽 W，框宽 w
总间距 = W - (N × w)
每间距 = 总间距 / (N + 1)
第 k 个框的 x = 左间距 + k × (w + 间距)
```

### 3. 连线对齐
- 箭头终点是否对准框体中心？
- 虚线与实线是否表达正确的关联强度？
- 跨层连线是否交叉凌乱？

### 4. 版本号
| 变更类型 | 版本变化 | 示例 |
|---------|---------|------|
| 新增/删除厂商层级、网络代际 | 大版本 +1 | v1→v2 |
| 标题修正、连线调整、标签更新 | 小版本 +1 | v2.0→v2.1 |
| 术语错误、拼写修正 | 小版本 +0.1 | v2.1→v2.1.1 |

## 输出文件

每次更新后同步生成并保持版本号一致：
```
[project-name]-knowledge-graph.html  (源文件)
[project-name]-knowledge-graph.svg   (独立矢量图)
```

版本号标注在副标题中，如 `v2.1 · 2026-06-10`。
