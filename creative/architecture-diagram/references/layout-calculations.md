# 虚线边界内框体均匀分布计算

## 核心公式

给定虚线边界 `x_b` 和 `width_b`，在边界内放置 N 个宽度相同的内框 `width_i`：

```
可用空间 = width_b
总框宽    = N × width_i
总间隙    = width_b - (N × width_i)
单侧间隙  = 总间隙 / (N + 1)        ← 左间隙 = 右间隙 = 框间间隙

第 k 个框的 x 坐标 (k = 0 .. N-1)：
  x_k = x_b + 单侧间隙 + k × (width_i + 单侧间隙)
```

验证公式：
```
最后一个框右边缘 = x_b + 单侧间隙 + (N-1) × (width_i + 单侧间隙) + width_i
                 = x_b + 单侧间隙 + N × width_i + (N-1) × 单侧间隙
                 = x_b + N × width_i + N × 单侧间隙
                 = x_b + width_b                                    ✓
```

## 常见错误

### 错误1：先放框再调边界

❌ 先画了 4 个框，然后扩大虚线框 → 只把最后一个框右移
→ 框间间距不均匀

✅ 每次边界尺寸变化，**从头重新计算所有内框 x 坐标**

### 错误2：长宽不一的框放在同一行

❌ 第一框宽 200、第二框宽 180、第三框宽 130
→ 无法实现均匀分布，因为 `(width_b - sum(width_i)) / (N+1)` 计算出的间隙虽然相等，但框体本身不对称看起来仍不均衡

✅ **同一行的所有框必须等宽**。如果宽度不同，调整到统一宽度或走两行布局

### 错误3：增量式调整

❌ `这个框靠左了，x 加 5px` → 破坏整行均匀性
✅ 重新用公式计算整行

## 示例数据库

用户交互过程中各版本的边界和内框坐标记录（仅作参考，不要直接套用——下次要重新算）：

### 最终版本 (V3)
- 虚线框: x=225, width=645, 右边界=870
- 第一行 4 框 (各 130): x = 250, 405, 560, 715
  - 单侧间隙 = (645 - 4×130) / 5 = 125/5 = 25
  - 验证: 715 + 130 + 25 = 870 ✓
- 第二行 3 框 (各 150): x = 274, 473, 672
  - 单侧间隙 = (645 - 3×150) / 4 = 195/4 = 48.75 → 取整 49
  - 验证: 672 + 150 + 48 = 870 ✓ (取整误差 1px)
