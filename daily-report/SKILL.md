# 运维开发日报

生成适用于运维/开发人员的专业日报。

## 核心公式

今日完成 + 明日计划 + 需支持/风险

## 撰写原则

1. **结论先行**：开头直接说"今日完成X项任务"
2. **动作+结果+数据**：每项工作用「动词+结果+数据」表达
3. **量化表达**：巡检10台、修复3个Bug、部署2个服务
4. **简洁专业**：150-200字，即看即懂

## 输出格式（在线提醒）

当直接回复用户日报提醒时，使用以下可读格式：

```
【今日日报提醒】- {日期}

共记录 {N} 项工作，合计约 {Xh} 可追踪工时。

{结构化表格或列表}
```

## 岗位模板

### 运维
- 服务器巡检：范围、结论、有无异常
- 故障处理：类型、处理时长、结果
- 脚本执行/服务维护
- 监控告警：告警量、处理情况

### 开发
- 功能开发：接口/页面/模块、代码量、是否达提测标准
- Bug修复：数量、影响范围、已回归验证
- 技术调研：内容、结论
- 会议/沟通：主题、产出

## ⚠️ 用户自定义实现

此技能为通用模板。本会话的用户已通过 `productivity/daily_report` 技能实现了完整的自定义日报工作流，**请优先加载 `daily_report` 而非此技能**。差异：

| 维度 | 本技能（通用） | `daily_report`（用户定制） |
|------|--------------|--------------------------|
| 文件格式 | 〖今日完成〗分段文本 | pipe-delimited 流水账（`日期\|项目\|工时\|内容`） |
| 存储路径 | 无 | `~/Documents/Obsidian Vault/工作/日报/YYYYMMDD.md` + `~/knowledge/工作/日报/` symlink |
| 前置聚合 | 无 | `daily-report-prep --date YYYYMMDD` 本地脚本 |
| 后置步骤 | 无 | `kb-search.py refresh` 刷新索引 |
| 必填区段 | 无 | 软件项目 / 日常维护 / 项目部署（三个区段不可省略） |

## cron 任务响应格式

作为 cron 作业运行时（无用户在场），完整流程：

1. 运行 `daily-report-prep --date YYYYMMDD` 尝试聚合日志
2. 若返回无数据 → 用 `session_search` 搜索当天会话获取工作记录
3. 按 pipe-delimited 格式写入 `~/Documents/Obsidian Vault/工作/日报/YYYYMMDD.md`
4. 确保 `~/knowledge/工作/日报/YYYYMMDD.md` 存在且内容一致
5. 运行 `kb-search.py refresh` 刷新索引
6. 输出结构化提醒（表格/列表 + 是否需用户补充确认的问询）
7. 若当天无工作记录 → 输出 `[SILENT]`
