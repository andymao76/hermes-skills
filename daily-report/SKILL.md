# 运维开发日报

生成适用于运维/开发人员的专业日报。

## 核心公式

今日完成 + 明日计划 + 需支持/风险

## 撰写原则

1. **结论先行**：开头直接说"今日完成X项任务"
2. **动作+结果+数据**：每项工作用「动词+结果+数据」表达
3. **量化表达**：巡检10台、修复3个Bug、部署2个服务
4. **简洁专业**：150-200字，即看即懂

## 输出格式

```
【今日日报】- {日期} - {姓名}

〖今日完成〗（按优先级）
{动词+事项+结果+数据}，{附带效果说明}
...

〖异常情况〗
今日无异常 / {问题描述} | 处理状态：已解决/处理中

〖明日计划〗
{具体可量化的工作目标}
```

## 岗位模板

### 运维
- 服务器巡检：范围、结论、有无异常
- 故障处理：类型、处理时长、结果
- 脚本执行/服务维护
- 监控告警：告警量、处理情况

### 开发
- 功能开发：接口/页面/模块、代码量、是否达提测标准
- Bug修复：数量、影响范围、已回归验证
- 技术调研：内容、结论
- 会议/沟通：主题、产出

## 触发词

写日报、生成日报、今日日报、daily report
