# 中文头条新闻聚合提示词模板

可复用于 `cronjob` 的 AI 驱动新闻日报任务。

## 完整提示词

```
你的任务是生成一份中文头条新闻日报。

时间范围：从昨天 00:00 到今天 09:30（北京时间 UTC+8）的新闻。

步骤：
1. 使用 web_search 搜索过去24小时内的头条新闻，关键词用中文如"今日头条新闻"、"热点新闻"、"国内外要闻"等，多搜几次覆盖不同领域（国际、国内、科技、财经、社会）
2. 从搜索结果中精选 8-10 条最重要的新闻
3. 用 web_extract 提取其中 3-5 条重点新闻的详细内容
4. 整理成以下格式的日报：

---
## 📰 每日头条新闻 | {日期}

### 🔥 今日要闻
| # | 标题 | 来源 |
|---|------|------|
| 1 | xxx | xxx |

### 📋 详细摘要
每条重点新闻用 2-3 句话概括要点

### 💡 今日关键词
3-5 个今日热点关键词
---

要求：
- 全部用简体中文
- 新闻真实可查，带上来源
- 格式清晰，适合在 Telegram/Discord 阅读
- 如果某条新闻无法提取详情，标注"（摘要基于搜索片段）"
```

## 参数配置

- `enabled_toolsets`: `["web"]` — 只需搜索和提取
- `deliver`: `telegram,discord` — 避免微信频率限制
- `schedule`: 按需设置，如 `30 9 * * *`（每天 9:30）

## 变体

可根据需求调整：
- **时间窗口**: 修改提示词中的时间范围描述。当用户指定精确范围（如"从昨天 00:00 到今天 09:30"），直接嵌入原文而非用"24小时"
- **领域聚焦**: 将搜索关键词限定在特定领域（如"科技新闻"、"财经要闻"）
- **语言**: 保持中文输出，或改为其他语言
- **格式**: 可调整为纯表格或列表格式，适配不同平台

## 与 Script 模式 cron 共存

当系统同时运行 script 模式（如 GitHub Trending）和 AI 模式（头条新闻）两个 cron 时，注意：
- script 模式无 LLM 调用，零 token 成本，适合固定数据源的格式化输出
- AI 模式有搜索+提取开销，但内容更灵活多样
- 两者可以设置不同投递平台，或投递到同一平台的不同话题/频道
