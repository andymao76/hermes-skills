# 知识隔离与分析 — 本质理解

> 本文件记录核心概念辨析：Hermes Agent 如何向在线 LLM 发送知识，以及"发送"与"训练"的根本区别。

## 核心问题

```
用户知识
    ↓
Hermes Prompt
    ↓
在线 LLM API
    ↓
返回答案
```

**不等于：**

```
用户知识
    ↓
重新训练 LLM 模型权重
```

## 关键认知

### 1. Hermes 的工作方式

当用户查询知识库时：

```
知识库 (knowledge/)
  ↓
RAG 检索（找到相关段落 ~1000-5000 字）
  ↓
拼接到 System Prompt
  ↓
发送给在线 LLM API
  ↓
模型读取 Prompt 内容 → 产生回答
```

知识库中的内容**不会被用于重新训练模型**，只是每一次请求时被临时拼入 Prompt。

### 2. "看到"与"学会"的区别

| 概念 | 说明 |
|------|------|
| **看到了** ✅ | LLM 本次请求读取了 Prompt 中的知识片段来回答 |
| **学会了** ❌ | 知识进入模型权重，变成模型参数的一部分 |

在线 LLM API（DeepSeek / OpenAI / Anthropic / SiliconFlow）默认：
- API 数据 **不用于模型训练**
- Prompt 和 Response 在会话结束后即丢弃
- 但服务商理论上可以 **看到** 每次 API 调用中的 Prompt 内容

### 3. 数据存储 ≠ 数据推理

```
本地存储              本地推理
   ↓                   ≠
知识在本地磁盘上       模型在本机运行
```

知识文件在 `knowledge/` 目录下是本地存储，但：
- 如果 RAG 检索到了相关内容
- 通过 API 发给了在线 LLM
- 那数据就**离开本地了**

## 风险矩阵

| 场景 | 数据进 Prompt？ | 进模型训练？ |
|------|:--------------:|:-----------:|
| 本地模型（Ollama/Qwen GGUF） | ❌ 不进 Prompt | ❌ 数据不出本地 |
| API 调用（DeepSeek/OpenAI） | ✅ 进 Prompt | ❌ 不用于训练 |
| 网页版 Chat 对话 | ✅ 进 Prompt | ⚠️ 可能用于训练 |
| 企业私有部署 | ❌ 不进外部 Prompt | ❌ 完全可控 |

## 结论

- API 调用不会让模型"学会"你的私有知识
- 但每次调用的 Prompt 内容会被服务商接触到
- 因此：**客户机密、监听数据、密码和内部资料应当严格隔离或脱敏**
- 等私有 LLM 部署后可安全导入全部私有领域知识
