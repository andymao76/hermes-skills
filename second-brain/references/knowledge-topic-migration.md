# 主题知识迁移工作流

从知识库中散落于多个目录的同主题文件提取到聚焦子目录。

## 适用场景

当 `编程规范/`、`skills/`、`AI_Agent相关/`、`research/`、`notes/`、`bigdata/`、`数据库相关/` 等不同目录中散落着同一题材的文档时，迁移到一个聚焦目录（如 `program-info/`）。

## 工作流

1. **扫描摸底** — 多关键词搜索目标题材文件（`*python*`, `*mcp*`, `*sql*`, `*git*`, `*docker*`）
2. **目录结构摸底** — `find ~/knowledge -maxdepth 2 -type f | sort` 确认哪些目录含目标内容
3. **确定范围** — 读取候选文件确认归属，排除领域性太强的内容
4. **创建目标结构** — `mkdir -p target/{子目录1,子目录2,...}`
5. **复制文件** — `cp` 从原始位置到目标，保持文件名
6. **创建 `_index.md`** — 每个子目录及总目录都创建含 wikilinks 的索引
7. **原位置替换为跳转指引** — 每个原始文件改为：
   ```
   > **此文档已迁移至 [[target/子目录/文件名]]**
   ```
8. **enzyme refresh** — 重建搜索索引

## 关键原则

- 保留原始文件为跳转指引（不 mv 删除）
- 每个子目录必须含 `_index.md`
- 每子目录 1-6 个文件为宜
- 迁移后检查 refresh 输出的 `files_new` 数量
